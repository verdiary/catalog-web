# AGENTS.md - Информация для агентов ИИ

## Обзор проекта

**Название проекта:** Справочник садовода  
**Тип проекта:** Статический сайт на MkDocs с темой Material  
**Назначение:** Справочник по растениям (овощи, плодовые деревья, цветы) с подробной агротехнической информацией

## Структура проекта

```
catalog-web/
├── mkdocs.yml                 # Конфигурация MkDocs
├── docs/                      # Корень документации
│   ├── index.md               # Главная страница
│   ├── shablon.md             # Шаблон для создания новых страниц растений
│   ├── ovoshhi/               # Категория: Овощи
│   │   ├── tomat.md           # Томат 'Бычье сердце'
│   │   ├── ogurec.md          # Огурец 'Изящный'
│   │   └── perec.md           # Перец 'Калифорнийское чудо'
│   ├── plodovye/              # Категория: Плодовые деревья
│   │   └── yablonya.md        # Яблоня 'Антоновка'
│   └── cvety/                 # Категория: Цветы
│       └── roza.md            # Роза 'Пьер де Ронсар'
├── dev-docs/                  # Документация по разработке
└── venv/                      # Виртуальное окружение Python (обязательно!)
```

## Настройка окружения

### ВАЖНО: Использование виртуального окружения

Проект использует Python и MkDocs, поэтому **обязательно** необходимо использовать виртуальное окружение для изоляции зависимостей.

#### Создание виртуального окружения (если еще не создано):

```bash
python -m venv venv
```

#### Активация виртуального окружения:

**Для macOS/Linux:**
```bash
source venv/bin/activate
```

**Для Windows:**
```bash
venv\Scripts\activate
```

#### Установка зависимостей:

```bash
pip install mkdocs mkdocs-material
```

#### Деактивация виртуального окружения (после завершения работы):

```bash
deactivate
```

## Работа с проектом

### Запуск локального сервера для разработки:

```bash
# Сначала активируйте виртуальное окружение!
source venv/bin/activate

# Запустите сервер разработки
mkdocs serve
```

Сайт будет доступен по адресу: http://127.0.0.1:8000/

### Сборка статического сайта:

```bash
# Сначала активируйте виртуальное окружение!
source venv/bin/activate

# Соберите сайт
mkdocs build
```

Результат будет в папке `site/`, которую можно развернуть на любом хостинге.

### Добавление новых растений:

1. Скопируйте шаблон `docs/shablon.md`
2. Переименуйте копию в соответствии с названием растения
3. Разместите файл в соответствующей категории (ovoshhi, plodovye, cvety)
4. Заполните информацию о растении
5. Добавьте новый файл в навигацию в `mkdocs.yml`

## Особенности проекта

### Настройки MkDocs

- Тема: Material for MkDocs
- Поддержка светлой и темной темы с автоматическим переключением
- Включенные расширения:
  - admonition (для блоков [!tip] и [!warning])
  - tables (поддержка таблиц)
  - pymdownx.emoji (поддержка эмодзи)

### Структура страницы растения

Каждая страница растения содержит следующие разделы:
1. Название сорта с эмодзи
2. Характеристики сорта и использование
3. Календарь работ (в виде таблицы)
4. Пошаговая агротехника
5. Важные нюансы для региона и сорта (в виде блоков [!tip] и [!warning])

## Правила для агентов

1. **Всегда используйте виртуальное окружение** перед выполнением любых команд MkDocs.
2. При добавлении новых растений следуйте шаблону `docs/shablon.md`.
3. Не забывайте обновлять навигацию в `mkdocs.yml` при добавлении новых страниц.
4. Проверяйте результат работы с помощью `mkdocs serve` перед коммитом изменений.
5. Следите за согласованным стилем оформления всех страниц растений.

## Полезные команды

```bash
# Активация виртуального окружения
source venv/bin/activate

# Запуск сервера разработки
mkdocs serve

# Сборка сайта
mkdocs build

# Проверка конфигурации
mkdocs build --strict

# Создание новой страницы (пример)
cp docs/shablon.md docs/ovoshhi/novyy_rastenie.md
```
